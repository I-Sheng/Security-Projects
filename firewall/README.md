# Firewall

This module covers Linux firewall implementation from the ground up: writing kernel modules, hooking into the Netfilter framework, configuring stateless and stateful `iptables` rules, rate limiting, and DNAT-based load balancing.

> **Source:** SEED Labs — Firewall Exploration Lab (Wenliang Du)

## Prerequisites

- Linux host with kernel headers installed (`apt install linux-headers-$(uname -r)`)
- Root / `sudo` privileges
- `iptables`, `conntrack` available
- Docker (for the multi-host lab topology)

## Lab Environment

Containers on two subnets:

| Host | IP | Role |
|---|---|---|
| Attacker | `10.9.0.1` | Attack source |
| Host A | `10.9.0.5` | External host |
| Router | `10.9.0.11` / `192.168.60.11` | Routes between subnets; firewall rules applied here |
| Internal Host 1 | `192.168.60.5` | Protected server |
| Internal Host 2 | `192.168.60.6` | Protected server |
| Internal Host 3 | `192.168.60.7` | Protected server |

## Task 1 — Implementing a Firewall with Netfilter

### Task 1A — Simple Kernel Module

A minimal kernel module that prints to the kernel log on load and unload:

```c
#include <linux/module.h>
#include <linux/kernel.h>

int initialization(void) { printk(KERN_INFO "Hello World!\n"); return 0; }
void cleanup(void)        { printk(KERN_INFO "Bye-bye World!.\n"); }

module_init(initialization);
module_exit(cleanup);
```

**Makefile:**

```makefile
obj-m += hello.o
all:
    make -C /lib/modules/$(shell uname -r)/build M=$(PWD) modules
clean:
    make -C /lib/modules/$(shell uname -r)/build M=$(PWD) clean
```

Compile, load, and verify:

```bash
make
sudo insmod hello.ko
lsmod | grep hello
sudo rmmod hello
dmesg    # shows "Hello World!" on load and "Bye-bye World!" on unload
```

**Sub-task 3 — Implement `blockTelnet` and `blockICMP`:**

Prevent other machines from pinging or telneting into the router (`10.9.0.1`). Implement two separate hook functions registered to the same hook point:

```c
// Block TCP to 10.9.0.1:23
unsigned int blockTelnet(void *priv, struct sk_buff *skb,
                         const struct nf_hook_state *state) {
    struct iphdr *iph = ip_hdr(skb);
    struct tcphdr *tcph;
    u32 ip_addr;
    in4_pton("10.9.0.1", -1, (u8 *)&ip_addr, '\0', NULL);
    if (iph->protocol == IPPROTO_TCP) {
        tcph = tcp_hdr(skb);
        if (iph->daddr == ip_addr && ntohs(tcph->dest) == 23)
            return NF_DROP;
    }
    return NF_ACCEPT;
}

// Block ICMP to 10.9.0.1
unsigned int blockICMP(void *priv, struct sk_buff *skb,
                       const struct nf_hook_state *state) {
    struct iphdr *iph = ip_hdr(skb);
    u32 ip_addr;
    in4_pton("10.9.0.1", -1, (u8 *)&ip_addr, '\0', NULL);
    if (iph->protocol == IPPROTO_ICMP && iph->daddr == ip_addr)
        return NF_DROP;
    return NF_ACCEPT;
}
```

Test from `10.9.0.5`:
```bash
ping 10.9.0.1
telnet 10.9.0.1
```

> **Important:** Always unregister hooks in `removeFilter()` to avoid kernel crashes when the module is removed.

### Task 2B — Protecting the Internal Network

Topology: external hosts on `10.9.0.0/24`, internal hosts on `192.168.60.0/24`, router bridging both via `eth0` (external) and `eth1` (internal).

Enforce the policy that internal hosts can ping externally but external hosts cannot ping internal hosts directly:

```bash
iptables -P FORWARD DROP
iptables -A FORWARD -i eth1 -o eth0 -s 192.168.60.0/24 -p icmp --icmp-type echo-request -j ACCEPT
iptables -A FORWARD -i eth0 -o eth1 -d 192.168.60.0/24 -p icmp --icmp-type echo-reply   -j ACCEPT
iptables -A FORWARD -i eth0 -o eth1 -d 192.168.60.0/24 -p icmp --icmp-type echo-request -j DROP
iptables -A INPUT   -i eth0 -p icmp --icmp-type echo-request -j ACCEPT
iptables -A OUTPUT  -o eth0 -p icmp --icmp-type echo-reply   -j ACCEPT
iptables -P INPUT  DROP
iptables -P OUTPUT DROP
```

Requirements to verify:
1. Outside hosts **cannot** ping internal hosts.
2. Outside hosts **can** ping the router.
3. Internal hosts **can** ping outside hosts.
4. All other traffic between internal and external networks is blocked.

### Task 2C — Protecting Internal Servers

Using only stateless rules (no connection tracking), enforce:

1. Outside hosts can only Telnet to `192.168.60.5` (port 23), not to other internal hosts.
2. Outside hosts cannot access other internal servers.
3. Internal hosts can access all internal servers freely.
4. Internal hosts cannot access external servers.

```bash

# External → 192.168.60.5:23

# Return traffic
iptables -A FORWARD -i eth1 -o eth0 -s 192.168.60.5 -d 10.9.0.0/24 -p tcp --sport 23 -j ACCEPT
# Internal ↔ internal
iptables -A FORWARD -i eth1 -o eth1 -p tcp -j ACCEPT
```

## Task 3 — Connection Tracking and Stateful Firewall

### Task 3A — Experimenting with `conntrack`

Run `conntrack -L` on the router while conducting each experiment below. Describe your observations and explain how long each connection state is maintained.

```bash
watch -n 1 conntrack -L
```

- **ICMP:** `ping 192.168.60.5` from `10.9.0.5`
- **UDP:** `nc -lu 9090` on `192.168.60.5`; `nc -u 192.168.60.5 9090` from `10.9.0.5`
- **TCP:** `nc -l 9090` on `192.168.60.5`; `nc 192.168.60.5 9090` from `10.9.0.5`

*Observations:*
- **ICMP:** Request + reply grouped as one conntrack entry; both directions shown.
- **UDP:** Once traffic flows in both directions, the entry is marked `[ASSURED]` and persists ~2 minutes after the last packet.
- **TCP:** Full handshake tracked; state transitions `SYN_SENT` → `ESTABLISHED`; entry is `[ASSURED]` after bidirectional data and lingers ~2 minutes after teardown.

### Task 3B — Stateful Firewall Rules

Rewrite the Task 2C rules using `conntrack`, and add a rule allowing internal hosts to initiate TCP connections to any external server (which was not allowed in 2C):

```bash
iptables -P FORWARD DROP

# Allow new connections: external → 192.168.60.5:23 only
iptables -A FORWARD -i eth0 -o eth1 \
  -s 10.9.0.0/24 -d 192.168.60.5 -p tcp --dport 23 --syn \
  -m conntrack --ctstate NEW -j ACCEPT

# Allow new connections: internal → any external host
iptables -A FORWARD -i eth1 -o eth0 \
  -s 192.168.60.0/24 -d 10.9.0.0/24 -p tcp --syn \
  -m conntrack --ctstate NEW -j ACCEPT

# Allow packets belonging to established/related flows
iptables -A FORWARD -p tcp -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT
```

**Comparison:**

| Aspect | Stateless (Task 2C) | Stateful (Task 3B) |
|---|---|---|
| Rule matching | Per-packet header fields only | Tracks connection state across packets |
| Return traffic | Requires explicit bidirectional rules | Covered automatically by `ESTABLISHED,RELATED` |
| Rule count | Grows with each service | Fewer, higher-level rules |
| Overhead | No conntrack table | Small conntrack overhead, negligible for typical loads |

## Task 4 — Rate Limiting

Limit forwarded traffic from `10.9.0.5` to 10 packets per minute with a burst allowance of 5:

```bash
iptables -A FORWARD -s 10.9.0.5 -m limit --limit 10/minute --limit-burst 5 -j ACCEPT
iptables -A FORWARD -s 10.9.0.5 -j DROP
```

Run the experiment with and without the second rule, then explain why the second rule is necessary.

*Observation:* Without the second rule, packets that don't match the rate-limiter fall through to the default `ACCEPT` policy and are **not** dropped — the rate limit has no effect. The DROP rule is essential to catch the overflow.

## Task 5 — Load Balancing with DNAT

Start UDP servers on `192.168.60.5`, `.6`, and `.7` (port 8080): `nc -luk 8080`

Distribute incoming UDP traffic on port 8080 across the three backend servers.

**Round-robin (deterministic nth mode):**

```bash
iptables -t nat -A PREROUTING -p udp --dport 8080 \
    -m statistic --mode nth --every 3 --packet 0 -j DNAT --to-destination 192.168.60.5:8080
iptables -t nat -A PREROUTING -p udp --dport 8080 \
    -m statistic --mode nth --every 3 --packet 1 -j DNAT --to-destination 192.168.60.6:8080
iptables -t nat -A PREROUTING -p udp --dport 8080 \
    -m statistic --mode nth --every 3 --packet 2 -j DNAT --to-destination 192.168.60.7:8080
```

**Random (probabilistic mode):**

```bash
iptables -t nat -A PREROUTING -p udp --dport 8080 \
    -m statistic --mode random --probability 0.33 -j DNAT --to-destination 192.168.60.5:8080
iptables -t nat -A PREROUTING -p udp --dport 8080 \
    -m statistic --mode random --probability 0.5  -j DNAT --to-destination 192.168.60.6:8080
iptables -t nat -A PREROUTING -p udp --dport 8080 \
    -j DNAT --to-destination 192.168.60.7:8080
```

Send 50 test packets and report the distribution from both modes. Explain how the probability values were chosen for random mode, and explain any deviation from a perfect 1/3 split.

*Observation:* With only 50 test packets the observed distribution deviates from a perfect 1/3 split due to small sample size and UDP's unreliability. The `nth` mode gives strictly deterministic alternation; the `random` mode converges to the target probability only with large sample sizes.
